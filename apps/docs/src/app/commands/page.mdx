# Commands

## Zero-arg mode

```bash
portly
```

Runs the `"dev"` script from `package.json` through the proxy. The app name is inferred from the package name, git root, or directory name. From a monorepo root (detected via `pnpm-workspace.yaml` or `package.json` `"workspaces"`), starts all workspace packages that have the target script.

```bash
portly                      # single app: run dev script
portly                      # monorepo root: start all apps
portly --script start       # run "start" instead of "dev"
```

Use a `portly.json` to override defaults. See [Configuration](/configuration).

## Run an app

```bash
portly run [--name <name>] [cmd] [args...]
portly <name> <cmd> [args...]
```

`portly run` infers the project name from `portly.json`, `package.json`, git root, or directory name. When no command is given, runs the configured script (default: `"dev"`) from `package.json`. Use `--name` to override the inferred name while still applying worktree prefixes.

`portly <name>` uses an explicit name with no inference or prefixing.

```bash
portly run                             # run dev script through proxy
portly run next dev                    # infer name from project
portly run --name myapp next dev       # override inferred name
portly myapp next dev                  # explicit name
portly api pnpm start
portly docs.myapp next dev
```

## Get a service URL

```bash
portly get <name>
```

Print the URL for a service. Useful for wiring services together in scripts or env vars:

```bash
BACKEND_URL=$(portly get backend)
```

Applies worktree prefix detection by default. Use `--no-worktree` to skip it.

## Alias (static routes)

```bash
portly alias <name> <port>
portly alias <name> <port> --force
portly alias --remove <name>
```

Register a route for a service not managed by portly (e.g. a Docker container). Aliases persist across stale-route cleanup.

```bash
portly alias my-postgres 5432     # -> https://my-postgres.localhost
portly alias redis 6379           # -> https://redis.localhost
portly alias --remove my-postgres # remove the alias
```

## List routes

```bash
portly list
```

Shows active routes and their assigned ports.

## Trust the CA

```bash
portly trust
```

Adds the portly certificate authority to your system trust store. Required once for HTTPS with auto-generated certs.

## Clean up

```bash
portly clean
```

Stops the proxy if it is running, removes the portly CA from the OS trust store when portly installed it, deletes allowlisted files under `~/.portly`, the system state directory, and `PORTLY_STATE_DIR` when set, and removes the portly block from `/etc/hosts`. May prompt for elevated privileges. Custom `--cert` / `--key` paths are not removed.

## Prune orphans

```bash
portly prune
portly prune --force
```

Finds and kills orphaned dev server processes left behind by crashed portly sessions. When the CLI is killed with `kill -9` or crashes, child dev servers may survive and continue holding their ports. This command checks routes whose owning process is dead but whose port is still in use, terminates the orphans, and removes the stale route entries. Use `--force` to send SIGKILL instead of SIGTERM.

## OS startup service

```bash
portly service install
portly service install --lan
portly service install --wildcard
PORTLY_STATE_DIR=~/.portly-lan PORTLY_LAN=1 portly service install
portly service status
portly service uninstall
```

Installs the proxy as an OS startup service. The default is HTTPS on port 443 with `.localhost` names, but `service install` accepts proxy options including `--port`, `--no-tls`, `--lan`, `--ip`, `--tld`, `--wildcard`, `--cert`, and `--key`. Use `--state-dir <path>` or `PORTLY_STATE_DIR=<path>` to choose where service state and logs are written.

The chosen service configuration is written into launchd, systemd, or Task Scheduler and reused after reboot. `portly service status` reports the installed port, HTTPS mode, TLD, LAN mode, wildcard mode, and state directory. macOS and Linux use a root-owned service so port 443 can bind at boot. Windows uses a Task Scheduler startup task that runs as SYSTEM. Installation and removal may require administrator privileges. `portly clean` automatically removes the service.

### LAN mode

Access services from phones and other devices on the same WiFi via mDNS (`.local` domains):

```bash
portly proxy start --lan
portly proxy start --lan --https
portly proxy start --lan --ip 192.168.1.42   # manual IP override
```

Make it permanent by adding `export PORTLY_LAN=1` to your shell profile. Portly also remembers LAN mode via `proxy.lan`, so a stopped LAN proxy starts in LAN mode again unless you override it with `PORTLY_LAN=0` for that start.

Uses `dns-sd` on macOS (built-in) and `avahi-publish-address` on Linux (`avahi-utils`). Not supported on Windows.

**Framework notes:**

- **Next.js**: Add `allowedDevOrigins: ['myapp.local', '*.myapp.local']` to `next.config.js` so LAN mode requests still work when portly adds a git worktree prefix.
- **Vite / React Router / SvelteKit / Astro**: Handled automatically via `__VITE_ADDITIONAL_SERVER_ALLOWED_HOSTS`.
- **Expo / React Native**: Add `NSAllowsLocalNetworking` to your `app.json` for iOS ATS (see [Getting Started](/) for the `.localhost` equivalent using `NSExceptionDomains`):

```json
{
  "expo": {
    "ios": {
      "infoPlist": {
        "NSAppTransportSecurity": {
          "NSAllowsLocalNetworking": true
        }
      }
    }
  }
}
```

## Hosts

```bash
portly hosts sync     # add current routes to /etc/hosts
portly hosts clean    # remove portly entries from /etc/hosts
```

Auto-sync is on by default. Set `PORTLY_SYNC_HOSTS=0` to disable.

## Bypass portly

```bash
PORTLY=0 pnpm dev
```

Runs the command directly without the proxy.

## Info

```bash
portly --help
portly --version
```
